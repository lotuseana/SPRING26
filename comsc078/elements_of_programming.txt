Primitive expressions and statements are the most basic building blocks of Python. A good way to generalize primitive expressions is in their ability to hold meaning without having to define "context" elsewhere in the Python code; by default, Python will already understand these statements. For example, defining primitive variables would look like "x = 10", "x = True", or "x = "hi"". In contrast, something like "x = hello()" holds no meaning to Python without first defining the function hello(). 

By combining elements, simpler or more primitive Python elements can build to more complex ones. This usually manifests in the form of computing one thing after the other and being able to use updated values to perform more computations on top of it. For example, the infix expression "2 + 3 * 4 - 1" (not a string, just emphasizing the expression) will perform computations one after the other: the result of 3*4 is added to 2, and the result of the addition gets subtracted by 1. In general, for-loops and if-statements fall under this category, because they are able to perform a task iteratively depending on state, and evaluate logical sequences, respectively.

By abstracting elements, the programmer is able to assign meaning to terms that Python otherwise does not find meaning in. For example, the programmer can define the "class Cat", in which functions "c.meow()" and "c.play()" can be defined and then called. By default, Python does not know what a Cat is, and does not know how a cat is meant to meow() or play(); the user must first define them. Defining variables, functions, etc. all fall under the category of abstraction.
